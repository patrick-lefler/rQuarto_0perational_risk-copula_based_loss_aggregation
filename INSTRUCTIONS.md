# The Diversification Question — Copula-Based Aggregation of Operational Risk Losses — Project Instructions

## 1. Project Overview
- **Short Description:** Models dependence between five operational loss categories using copulas, comparing diversified joint-loss capital against the naive sum of standalone category VaRs to quantify the model risk embedded in the diversification assumption itself.
- **Description:** This project simulates ten years of synthetic operational loss history across five Basel event-type categories for NexaCore Financial Technologies using a compound Poisson-lognormal frequency-severity framework. Rather than modeling each category's tail in isolation, the analysis fits and compares five copula families (Gaussian, Student-t, Clayton, Gumbel, Frank) to the joint loss structure, selects the best-fitting dependence model by AIC, and uses it to simulate the joint annual loss distribution. The central comparison is between the diversified 99.9% VaR computed on this joint distribution and the additive sum of standalone category VaRs — the implicit assumption under a silo-by-silo capital approach. A secondary analysis shows how much the estimated diversification benefit itself shifts depending on which copula family is assumed, framing dependence-structure choice as a source of model risk in its own right.
- **Abstract:** *[Finalized 2026-06-24 with confirmed pipeline results; see full text below.]*

  Operational risk capital is typically estimated category by category and then summed, an approach that implicitly assumes every loss type peaks at once. This project tests that assumption using ten years of synthetic annual loss data across five categories for NexaCore Financial Technologies, built around a single engineered dependence channel: a systems outage that elevates downstream process failures within the same year. A bivariate copula fit to the one category pair carrying genuine dependence selects a Student-t copula at two degrees of freedom — modest but real co-movement, with a Kendall's tau of 0.16. Simulating the joint annual loss distribution puts diversified 99.9% Value-at-Risk at €17.5 million, against an additive sum of standalone category VaRs of €24.9 million, a 29.5% reduction; Expected Shortfall shows a comparable 33.8% reduction. Recomputing this benefit under four alternative copula families moves the estimate by roughly one percentage point, not because dependence is unimportant but because only one of five categories carries any modeled dependence at all. Most of the diversification benefit comes from summing independent risks, not from the specific copula chosen. The more useful board-level message is not which copula fits best, but how much of an aggregate capital figure's apparent precision rests on categories nobody modeled any relationship between.

- **Target audience:** Key risk stakeholders and/or board members (CRO, Operational Risk Committee)
- **Output format:** HTML
- **Render command:** `quarto render index.qmd`
- **Expected output path:** `output/index.html`
- **Data source(s):** Synthetic — generated by `scripts/generate_losses.R`. Monthly loss events simulated over a 10-year horizon (120 months) across 5 categories. Approximate record count: ~3,000–6,000 individual loss events pre-aggregation (category-dependent Poisson λ), aggregated to 10 annual joint observations per Monte Carlo path × 50,000 paths for the copula simulation stage.

```r
library(sysfonts)      # Font lookup/download backend for showtext
```

**Project-specific learning (2026-06-25):** `_brand.yml` font declarations only style the Quarto HTML page (CSS-rendered headings/body text); they do not extend to R's graphics device, which rasterizes ggplot2 charts independently. If `theme_brand()` requests a font family not installed as a system font on the rendering machine, text elements render as blank rather than falling back to a default font — and since `theme_brand()`'s base `text` element sets the family for everything, the failure hits every text element in the chart at once (axis labels, legend, data labels), while text Quarto renders outside the image (e.g. the figure caption) is unaffected. Fix: register the brand's Google Fonts directly via `showtext::showtext_auto()` + `sysfonts::font_add_google()` in Setup, so they're available to the graphics device regardless of local installation. Requires network access on first run (cached after). This applies to every future project using `theme_brand()` with non-system fonts, not just this one.

**Namespace hazard:** `dplyr::select()`, `dplyr::filter()`, `dplyr::rename()` require explicit qualification once `copula` and/or `VineCopula` are loaded — confirmed recurring pattern from prior projects (`evd`, `fitdistrplus`, `simmer`). Qualify defensively from the Setup chunk onward.

AIC comparison was not decisive: t(df=2) wins (AIC=0.365) but t(df=4), t(df=6), and Gumbel all sit within ~1 AIC unit, i.e. comparable statistical support at n=10 — this directly motivates the §3.3.5 sensitivity table rather than trusting one winner. Decision: retain t(df=2) as the primary selection — it is the best fit within the five families the project explicitly set out to compare — and document the Tawn finding in §3.3.3 as an acknowledged scope limitation (five standard families tested, not an exhaustive search) rather than expanding the candidate set or re-running the prior stage.
